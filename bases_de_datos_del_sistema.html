BASES DE DATOS DEL SISTEMA

Las bases de datos del sistema son bases de datos que se crean cuando se instala SQL Server y sirven para soportar varias de las funcionalidades del motor.

Sin estas bases de datos, SQL Server no sería capaz de ofrecer ciertos servicios.

Las bases de datos del sistema están alojadas en la carpeta System Databases, tal y como se ve en la imagen.

Como se puede ver en la imagen, son cuatro (4) las bases de datos del sistema:

  • Master
  • Model
  • Msdb
  • Tempdb

Vamos a proceder a explicar, brevemente, cual es el objetivo de cada una de estas bases de datos.

  • Master: es la base de datos más importante. Almacena los metadatos del sistema, es decir, almacena la información técnica referente a las bases de datos de  los usuarios. Si esta base de datos no existiera, SQL Server no sería capaz de gestionar las bases de datos creadas por los usuarios.
  • Model: Es una base de datos que sirve como plantilla inicial para cualquier base de datos que se cree, es decir, cuando se crea una base de datos, SQL Server automáticamente copia lo que hay en Model a esa base de datos que se está creando. ES el formato inicial de cualquier otra base de datos.
  • Msdb: Con esta base de datos, SQL Server es capaz de administrar las bases de datos distribuidas, creadas por los usuarios.
  • Tempdb: Es una base de datos que le sirve a SQL Server para almacenar, temporalmente, ciertos datos que necesita para ejecutar sus procesos. Esta base de datos sirve como una extensión de la memoria RAM, la diferencia es que esta base de datos está grabada en el disco duro.
