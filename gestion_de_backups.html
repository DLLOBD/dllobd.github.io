GESTION DE BACKUPS

Cómo hacer un backup en SQL Server?

Para explicar cómo hacer un backup en SQL Server, vamos a suponer una base de datos llamada CONSTRUCTORA.

El primer paso es darle clic derecho a la base de datos, luego en el menu emergente que salga escoger la opción TASKS, y en el otro menú escoger BACK UP, tal y como se muestra en la imagen.

Se abre la ventana que se muestra en la imagen. Debemos eliminar todo lo que aparezca donde está lo que tiene resaltado azul. Eso lo eliminamos con el botón REMOVE.

Cuando esto esté vacío, damos clic al botón ADD.

Se abre una pequeña ventana, donde le damos clic al botón que tiene los tres puntos, y que se encuentra al lado derecho.

Después de esto se abrirá una ventana donde diremos es qué carpeta queremos grabar el backup. Además, hay que darle nombre al archivo que contendrá el backup, tal y como se ve en la imagen.

Después de todo esto, nos devolvemos dandole clic a los botones de OK, que están en la parte inferior de cada ventana que abrimos.

Al final, en la ruta especificada en este paso, queda grabado el archivo del backup, con el nombre especificado.

Cómo restaurar un backup en SQL Server?

Para restaurar un backup qu tenemos hecho, damos clic derecho a DATABASES, en el menu emergente damos clic en RESTORE DATABASE, tal y como se muestra en la imagen.

Luego de eso, aparece la pantalla que hay en la imagen. Como el backup lo tenemos en un dispositivo (disco duro, USB, etc.), escogemos la opción DEVICE, y le damos clic al boton con los 3 puntos.

Después de eso, aparece una pequeña ventana donde le damos clic al botón ADD.

Aparece la pantalla mostrada. Ahí es importante cambiar el tipo de archivos que vamos a buscar, poniendo la opción en ALL FILES, tal y como lo muestra el recuadro azul de la imagen.

Luego de esto, buscamos y seleccionamos en el árbol de carpetas el backup hecho anteriormente, como se muestra en el recuadro rojo de la imagen.

Y nos empezamos a devolver, dándole clic a los botones OK de las ventana abiertas.

De esta forma, ya nos aparecerá, en el explorador de objetos de SQL Server, la base de datos restaurada.
